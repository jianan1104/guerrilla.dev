## Method 1: Configuration from code

This guide explains how to configure the settings file for use in the SDK. The settings file is essential for setting up the connection and interaction with your device.

### Example Configuration File

Here's an example of what your configuration file might look like:

<Tabs items={['SSH', 'Telnet', 'Serial']} defaultIndex={0}>
  <Tab>
    ```py
    config = {
      "name": "EDR-G9010",
      "type": "mxros",
      "protocol": "ssh",
      "port": 22,
      "host": "192.168.127.254",
      "username": "admin",
      "password": "moxa",
}
    ```
  </Tab>
  <Tab>
    ```py
    config = {
      "name": "EDR-G9010",
      "type": "mxros",
      "protocol": "telnet",
      "port": 23,
      "host": "192.168.127.254",
      "username": "admin",
      "password": "moxa",
}
    ```
  </Tab>
  <Tab>
    ```py
    config = {
      "name": "EDR-G9010",
      "type": "mxros",
      "protocol": "serial",
      "host": "/dev/ttyUSB0",
      "username": "admin",
      "password": "moxa",
}
    ```
  </Tab>
</Tabs>

### Parameter Descriptions

| Parameter | Description |
|-----------|-------------|
| `name`    | The name of the device. |
| `type`    | The type of device. Supported types are `mxros`, `mxnos`, and `linux`. If a different type is specified, the system will automatically create a `BaseDevice` class to control the device. |
| `protocol`| The connection protocol used, such as `ssh`, `telnet`, or `serial`. |
| `port`    |  The connection port. It varies based on the `protocol`. |
| `host`    | The IP address or device path of the device. For `serial` type, this should be the device path like `/dev/ttyUSB0` or `COM1`. For other types, this should be the IP address, like `192.168.127.254`. |
| `username`| The username for login. |
| `password`| The corresponding password for user. |
